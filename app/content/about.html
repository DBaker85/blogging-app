Main page

The main page works by connecting to the database and looking for posts. The posts are listed by most recent and if they contain more than 300 characters, They are sliced and a readmore link is added. I found that a post that fits in less than 300 characters should not need to have it's own page.

If none are available, it displays: no posts yet!

Category page

The category page retrieves all the posts by category. This is a dynamic routing that is based on the url sent.

app.get('/category/:url', function (req, res) {
   displayCategoryPage(req, res, req.params.url);
});

clicking a nodejs category link anywhere on the site sends a link : '/category/nodejs'. The ending slash is retreived and pushed to the function that searches posts by category and displays all of them.

Post page

The post page retrieves the page by searching for the urlslug of the post (this is created on creation of a post).

app.get('/post/:url', function(req, res){
  url = req.params.url;
  displaySinglePost(req,res,url);
});

The fetched url slug is unique to each post and is created on creation of a post (see the create post section for more info)

Themer

The themer is to have a little fun with js and themes.When a color is typed, the backend checks a big list of colors that are stored in the app, a data base is overkill as the list doesn't need to be modified by the app

var colorList = {"sky-blue":{"primary":"blue"},"blue":{"primary":"blue"}}

This returns a color and the Javascript swaps the active stylesheet with the requested color.

The color is then set in localstorage to keep the theme active whenever the user visits the site.

Create password

Once the password is validated at the front end, I generate a random salt by using node crypto.

The username and the salt are stored into the database along with the salty hash that is encrypted and iterated a lot of times.

login

The login works by retreiving the salt, the username and re-running the same encrytpion on that was run to create it. If the salty has is the same as the one stored a session is created and the user logged in.

Create post

When typing, a live preview is shown to the right and the typed text is also stored in local storage just in case

This is stored in the Mongo document.A task is then run to activate the category associated to the post and the local storage is cleared

Edit posts

When clicking on edit. The post is fetched by custom id and is populated into the an edit form that appears. The Title cannot be changed as this is used for the urlslug. Once the edit is done and submitted. A field is added to the mongo document stating the date the post was modified. (this is displayed on the front end).

For deleting the post. A simple confirmation message is popped up and once confirmed, the post is deleted using it's unique id.

Then an asynchronous task is run to activate the new category and deactivate the old category if no other posts have that category associated.

Tags

Tags are created and stored using a simple form. These are used for displaying category pages. There is a list of all categories on the site. The active and inactive status is automatically done when creating, editing or deleting a post.
